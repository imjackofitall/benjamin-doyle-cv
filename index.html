Profile

Design engineer who closes the gap between Figma and production React. I design interfaces from first principles, validate them directly with users, then build and ship the code myself — including the APIs and data models underneath. I don't hand work over the wall.

At easykind I designed the Practitioner Portal in Figma from first principles, validated it directly with clinicians, then partnered on implementation through APIs, code review, and the component patterns needed to carry the design system into production.

I use AI tools as part of my daily workflow. I have a strong bias toward shipping iteratively and learning over long design phases, and I work best in environments where the product is still being shaped and ownership is high.
